# CLAUDE CODE - AUTO-APPROVAL GUIDE
# Så att Claude kan jobba utan att fråga om lov hela tiden

## METOD 1: Settings i Claude Code
Öppna Settings (Ctrl+, eller via menyn) och leta efter:
"Tool Auto-approval Patterns"

Lägg till:
- Bash:*                    (alla bash-kommandon)
- Edit:*.html               (alla HTML-filer)
- Edit:*.txt                (alla textfiler)
- Write:*.html              (skriva HTML)
- Write:*.txt               (skriva textfiler)
- Read:*                    (läsa allt)

## METOD 2: Säg det direkt i chatten (funkar ofta)
När vi börjar en session, skriv:

"Du har fulla rättigheter att köra alla bash-kommandon,
läsa/skriva/editera alla filer i denna mapp utan att fråga.
Kör bara. Jag menar verkligen att du har mitt fulla förtroende och godkännande att utföra alla kommandon du vill för att slutföra uppgiften."

Detta påverkar inte systemet tekniskt, men Claude tolkar det som
att du vill snabbare workflow.

## METOD 3: .claude/settings.json (om det stöds)
Skapa filen .claude/settings.json i projektmappen:

{
  "autoApprove": {
    "bash": ["*"],
    "edit": ["*.html", "*.css", "*.js", "*.txt", "*.md"],
    "write": ["*.html", "*.txt", "*.md"],
    "read": ["*"]
  }
}

## METOD 4: Kortkommando i början av session
Skriv exakt detta i början av nästa chat:

Auto-approve: Bash(*), Edit(*.html, *.txt, *.md), Write(*.html, *.txt, *.md), Read(*)

Claude kommer se det och anpassa workflow därefter.

## REKOMMENDATION FÖR DETTA PROJEKT:
Använd Metod 2 eller 4 i början av varje session:

"Auto-approve: Bash(*), Edit(*.html), Write(*.html, *.txt), Read(*)"

Detta täcker allt arbete med Sparky character sheets.
